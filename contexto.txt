"Descrição
A gestão de serviços e preços das modelos deve ser redesenhada para ser feita diretamente no perfil de cada modelo via modal, em vez de uma seção global de programas. Cada modelo terá seus próprios serviços com durações e preços específicos, com possibilidade de criar novos serviços/durações inline.
Plano de Implementação

Preview
Contexto
Atualmente, a gestão de serviços funciona em duas etapas separadas: (1) criar categorias de serviço (programas) e durações na seção global de Programas; (2) no perfil da modelo, os serviços criados aparecem automaticamente. O problema é que isso não permite que cada modelo tenha serviços e durações específicos — se uma modelo faz anal apenas em 30min e 1h, as outras durações aparecem desnecessariamente.

A proposta é tornar a gestão de serviços atômica por modelo: dentro do perfil de cada modelo, o usuário adiciona serviços específicos com suas durações e preços. Os serviços e durações existentes são reutilizáveis (selecionados de uma lista global), mas novos podem ser criados inline sem sair do perfil da modelo.

O que precisa ser feito
Redesenhar a seção de Serviços e Preços no perfil da modelo: (1) botão 'Adicionar Serviço' que abre modal; (2) no modal, multi-select de serviços existentes (anal, oral, etc.) com opção de criar novo serviço; (3) seleção de duração(ões) para o serviço com opção de criar nova duração; (4) campo de preço por duração; (5) listagem dos serviços da modelo com edição e remoção individuais. A seção global de Programas pode ser mantida como gerenciamento da lista master, mas a associação com modelos é feita no perfil."

Contexto
O módulo de Modelos possui uma seção de Fotos e Vídeos que exibe dados mockados sem integração real com o Minio. As fotos e vídeos das modelos são essenciais para o negócio — são usadas para apresentação aos clientes durante o atendimento. O que precisa ser feito
Implementar CRUD completo de mídias no perfil da modelo: upload de fotos e vídeos para o Minio, listagem com visualização real das mídias, inativação de mídias (soft delete com flag ativo/inativo) e deleção permanente. Remover todos os dados mockados."

O perfil da modelo possui botões de 'Trocar Conexão' e 'Remover Conexão' para gerenciar a integração com WhatsApp. É necessário implementar a integração real com a Evolution API para que o cliente possa gerir as conexões de WhatsApp das modelos diretamente no sistema.

Contexto
Cada modelo possui um número de WhatsApp que é usado pela IA para conduzir os atendimentos. A gestão dessa conexão (criar, reconectar, remover) deve ser feita pelo próprio cliente (Rossi) diretamente no sistema, sem depender da equipe de desenvolvimento. A Evolution API é a solução escolhida para gerenciar as instâncias de WhatsApp.

Os botões de 'Trocar Conexão' e 'Remover Conexão' já existem na interface mas não estão integrados com a Evolution API. Esta task implementa essa integração.

O que precisa ser feito
Implementar a integração com a Evolution API para gerenciamento de conexões WhatsApp no perfil da modelo: (1) criar nova conexão (gerar QR Code para escaneamento); (2) verificar status da conexão (conectado/desconectado); (3) reconectar (gerar novo QR Code); (4) remover conexão. O usuário deve conseguir escanear o QR Code diretamente no sistema."


"Descrição
A seção de 'Dúvidas' no perfil da modelo (para registrar perguntas e respostas) foi testada com outros clientes e não funciona na prática — confunde os clientes e não vai para frente. A decisão é remover completamente essa seção do sistema.
Plano de Implementação
Editar
Preview
Contexto
A seção de 'Dúvidas' no perfil da modelo foi criada com a intenção de permitir o registro de perguntas frequentes e respostas ideais, funcionando como uma base de conhecimento editável pelo cliente. Porém, após testes com outros clientes, foi identificado que essa abordagem não funciona na prática — os clientes ficam confusos sobre como usar e a seção acaba sendo abandonada.

A decisão é que o conhecimento da IA deve ser gerido pela equipe de desenvolvimento/operação, não deixado como campo editável pelo cliente no sistema. Portanto, a seção deve ser removida completamente."
